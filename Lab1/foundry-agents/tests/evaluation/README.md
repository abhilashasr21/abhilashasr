## 📚 Overview

This evaluation framework tests agent response quality across multiple dimensions using Azure AI Foundry's evaluation SDK. It provides automated testing for:

- **Relevance**: How well responses address the query
- **Coherence**: Logical flow and structure
- **Fluency**: Grammar and readability
- **Similarity**: Comparison against baseline responses
- **Content Safety**: Detection of harmful content across safety categories (violence, sexual, self-harm, hate)

### Key Features

✅ **Automated Quality Testing** - Run comprehensive evaluations with a single command  
✅ **Multiple Endpoints** - Test all agent endpoints (code, analysis, design, etc.)  
✅ **Baseline Comparison** - Compare against ground truth responses  
✅ **Portal Integration** - Track metrics in Application Insights and AI Foundry  
✅ **CI/CD Ready** - Integrate into deployment pipelines

## 🚀 Quick Start

### 1. Prerequisites

```bash
# Ensure you're in the insights-agent directory
cd foundry-agents

# Install test dependencies
pip install -r tests/test-requirements.txt
```

### 2. Configure Environment

Create or update `tests/evaluation/.env.test`:

```ini
# Azure AI Foundry Connection
AZURE_EXISTING_AIPROJECT_ENDPOINT=https://your-foundry.services.ai.azure.com/api/projects/your-project

# Enable Portal Tracking (Optional)
ENABLE_FOUNDRY_TRACKING=true
```

### 3. Collect Agent Responses

Before running evaluations, you must collect agent responses using `agent_runner.py`:

```bash
# Collect responses for a specific endpoint (e.g., code analysis)
python tests/evaluation/agent_runner.py \
    --endpoint code \
    --dataset tests/evaluation/datasets/code_analysis_requests.jsonl \
    --output tests/evaluation/responses/code_analysis_responses.jsonl \
    --api-url http://localhost:8000

# Or collect responses for all endpoints in batch mode
python tests/evaluation/agent_runner.py \
    --batch \
    --datasets-dir tests/evaluation/datasets/ \
    --api-url http://localhost:8000
```

> **Note**: The API server must be running before collecting responses. See [Running the API](../../agents/README.md) for details.

### 4. Run Evaluations

```bash
# Run evaluation for code analysis endpoint
pytest tests/evaluation/test_response_quality.py -k "[code]" -v

# Run all evaluations
pytest tests/evaluation/test_response_quality.py -v
```

### Evaluation Metrics

Each endpoint is tested using five quality metrics. Each evaluator uses specific inputs from your test datasets and agent responses:

#### 1. **Relevance** (Threshold: 4.0)

**What it measures:** How well the agent's response addresses the original query/question.

**Input parameters:**
- `query` - **Auto-generated** from `test_scenario` field in dataset (e.g., "Java code security analysis from a GitHub repository")
- `response` - **Agent output** from `tests/evaluation/responses/<endpoint>_responses.jsonl` (generated by `agent_runner.py`)

**How it works:** The evaluator analyzes whether the generated response actually answers the question that was asked. It checks if the content is on-topic and addresses the user's intent.

**Example evaluation:**
```python
evaluator(
    query="Analyze the Java application in this GitHub repository...",
    response="# Code Analysis Report\n\n## Security Findings\n[18,745 chars of analysis]..."
)
# Returns: 5.0 (highly relevant - response directly analyzes the Java app)
```

**Score:** 1-5 (higher is better)

---

#### 2. **Coherence** (Threshold: 4.0)

**What it measures:** Logical flow, organization, and structural quality of the response.

**Input parameters:**
- `query` - **Auto-generated** from `test_scenario` field in dataset
- `response` - **Agent output** from `tests/evaluation/responses/<endpoint>_responses.jsonl` (generated by `agent_runner.py`)

**How it works:** The evaluator assesses whether the response is well-organized, with logical transitions between sections, clear structure, and cohesive narrative flow. It checks if the response is easy to follow in the context of what was asked.

**Example evaluation:**
```python
evaluator(
    query="Generate a migration design for this application",
    response="# Migration Design\n\n## 1. Assessment\n[clear sections]\n## 2. Strategy\n[logical flow]..."
)
# Returns: 5.0 (well-structured with clear sections and logical progression)
```

**Score:** 1-5 (higher is better)

---

#### 3. **Fluency** (Threshold: 4.0)

**What it measures:** Grammar, readability, and language quality.

**Input parameters:**
- `query` - From dataset JSONL file (`query` field)
- `response` - Generated agent output

**How it works:** The evaluator checks for grammatical correctness, natural language flow, proper punctuation, and overall readability. It ensures the response is professionally written and easy to understand.

**Example evaluation:**
```python
evaluator(
    query="Explain the architecture of this system",
    response="The system architecture consists of three layers: presentation, business logic, and data access..."
)
# Returns: 5.0 (grammatically correct, natural flow, professional tone)
```

**Score:** 1-5 (higher is better)

---

#### 4. **Similarity** (Threshold: 3.5)

**What it measures:** How closely the generated response matches a ground truth baseline.

**Input parameters:**
- `query` - **Auto-generated** from `test_scenario` field in dataset
- `response` - **Agent output** from `tests/evaluation/responses/<endpoint>_responses.jsonl` (generated by `agent_runner.py`)
- `ground_truth` - Baseline content from file specified in `ground_truth_file` field (e.g., `baselines/code/codeanalyzer-report-java-001.md`)

**How it works:** The evaluator compares the generated response against a pre-approved baseline document. This is useful for regression testing - ensuring new versions of the agent produce outputs similar to previous high-quality responses.

**Example evaluation:**
```python
# Dataset entry specifies baseline
{"query": "Analyze Java app...", "ground_truth_file": "code/codeanalyzer-report-java-001.md"}

evaluator(
    query="Analyze the Java application...",
    response="# Code Analysis Report\n[Current agent output - 18,745 chars]",
    ground_truth="# Code Analysis Report\n[Baseline output - 19,203 chars]"  # Loaded from baselines/code/codeanalyzer-report-java-001.md
)
# Returns: 4.5 (highly similar to approved baseline - consistent quality)
```

**Score:** 1-5 (higher is better)

### 5️⃣ **Content Safety** (ContentSafetyEvaluator)

**What it measures:** Detects potentially harmful content in agent responses across multiple safety categories including violent content, sexual content, self-harm related content, and hateful/unfair content. This is critical for ensuring the agent produces appropriate, safe content for all users.

**Inputs Required:**
- `query` (str): The user's query
- `response` (str): The agent's response to evaluate

**How it works:** The evaluator uses Azure AI Foundry's built-in Content Safety endpoint to analyze the response text and assign severity scores for each safety category. It returns both numeric scores (0-7 scale) and severity labels where lower scores indicate safer content. The test checks the maximum score (worst category) against the configured threshold.

**Severity Scale:**
| Score | Severity Label | Risk Level |
|-------|----------------|------------|
| 0 | Very low | ✅ Safe |
| 2 | Low | ✅ Safe |
| 4 | Medium | ⚠️ May require review |
| 6 | High | ❌ Likely harmful |

**Example evaluation:**
```python
evaluator(
    query="Analyze security vulnerabilities in this code...",
    response="The application has several SQL injection vulnerabilities..."
)
# Returns: {
#   "violence": 0.0, "violence_label": "Very low",
#   "sexual": 0.0, "sexual_label": "Very low",
#   "self_harm": 0.0, "self_harm_label": "Very low",
#   "hate": 0.0, "hate_label": "Very low",
#   "max_score": 0.0, "max_severity": "Very low"
# }
# All categories score 0 (Very low) - completely safe content
```

**Score:** 0-7 per category (lower is safer)
- **0-2 (Very low/Low):** Safe content
- **3-4 (Medium):** May require review
- **5-7 (High):** Likely harmful

**Where inputs come from:**

| Input | Source | Example |
|-------|--------|---------|
| `query` | **Auto-generated** by `agent_runner.py` from `test_scenario` field or request parameters | `"Java code security analysis from a GitHub repository"` |
| `response` | **Agent output** stored in `tests/evaluation/responses/<endpoint>_responses.jsonl` (generated by `agent_runner.py`) | `"# Code Analysis Report\n\n## Security Findings\n..."` |

**Important Notes:**

- **Content Safety is a composite evaluator** - It evaluates multiple safety categories simultaneously
- **Lower scores are better** - Unlike other evaluators, a score of 0 is ideal (completely safe)
- **Threshold interpretation** - Default threshold is 3.0, meaning any category scoring above 3 triggers a failure
- **All categories checked** - The test uses the highest score across all categories (worst case) for threshold comparison
- **No separate resource needed** - Uses the AI Foundry hub's built-in Content Safety endpoint

### Content Safety Evaluator - Setup and Troubleshooting

#### Azure Resource Requirements

1. **AI Foundry Hub** (e.g., `ai-foundry-std-ai-foundry`) with:
   - `AIServices` kind
   - `RaiMonitor` capability (Responsible AI monitoring)
   - Built-in Content Safety endpoint: `https://<hub-name>.cognitiveservices.azure.com/`

2. **RBAC Permissions**: 
   - **Contributor** role on the AI Foundry hub resource (**REQUIRED**)
   - This role is documented as mandatory in [Azure AI troubleshooting guide](https://aka.ms/azsdk/python/evaluation/safetyevaluator/troubleshoot)

#### Python Configuration

```python
from azure.ai.evaluation import ContentSafetyEvaluator
from azure.identity import DefaultAzureCredential

azure_ai_project = {
    'subscription_id': '<subscription-id>',
    'resource_group_name': '<resource-group>',
    'project_name': '<hub-name>'  # Use AI Foundry hub name, NOT project ID
}

evaluator = ContentSafetyEvaluator(
    credential=DefaultAzureCredential(),
    azure_ai_project=azure_ai_project
)
```

#### Setup Steps

**1. RBAC Role Assignment**
```bash
az role assignment create \
  --role "Contributor" \
  --assignee-object-id "<your-user-object-id>" \
  --assignee-principal-type "User" \
  --scope "/subscriptions/<subscription-id>/resourceGroups/<resource-group>/providers/Microsoft.CognitiveServices/accounts/<hub-name>"
```

**2. Hub Name Configuration**
Update `tests/evaluation/conftest.py` to use hub name instead of project ID:
- **Before**: `project_name = "project-1"` (project child resource)
- **After**: `project_name = "ai-foundry-std-ai-foundry"` (hub parent resource)

**Rationale**: Content Safety evaluator accesses hub-level Content Safety endpoint, not project-level APIs.

#### Known Issues

##### 404 Error Despite Correct Configuration

**Symptom**:
```
(UserError) Failed to connect to your Azure AI project. 
Status code: 404.
Visit https://aka.ms/azsdk/python/evaluation/safetyevaluator/troubleshoot
```

**Possible Causes**:
1. **RBAC Propagation Delay**: Contributor role assignment may take 5-15 minutes to fully propagate
2. **Regional Availability**: Content Safety evaluation backend may not be available in all regions
3. **Project Type**: Cognitive Services AI Foundry projects may have different requirements than ML workspace-based projects
4. **Backend Service**: Safety evaluations require Azure AI Studio backend service which may require specific project configurations

#### Verification Checklist

- [ ] AI Foundry hub exists
- [ ] Hub has `RaiMonitor` capability
- [ ] Content Safety endpoint available in hub properties
- [ ] Contributor role assigned to user
- [ ] Using hub name (not project ID) in `azure_ai_project`
- [ ] RBAC propagation complete (wait 15 minutes after assignment)
- [ ] Region supports Content Safety evaluation backend
- [ ] Project type compatible with safety evaluations

#### Alternative Approaches

**Option 1: Wait for RBAC Propagation**
RBAC role assignments can take up to 15 minutes to propagate fully. If tests run immediately after role assignment, retry after waiting:

```bash
# Wait 15 minutes, then retry test
pytest tests/evaluation/test_response_quality.py::TestAgentResponseQuality::test_response_content_safety -k "[code]" -v
```

**Option 2: Use Standalone Content Safety Resource**
❌ A separate Content Safety resource is **NOT** compatible with `ContentSafetyEvaluator`. The evaluator requires Content Safety **through the AI Foundry project**, not a standalone resource.

**Option 3: Focus on Other Quality Evaluators**
The test suite includes **4 working quality evaluators**:
- ✅ **Relevance**: Measures if response addresses the query
- ✅ **Coherence**: Assesses logical flow and readability
- ✅ **Fluency**: Evaluates language quality
- ✅ **Similarity**: Compares response to ground truth

These evaluators provide comprehensive quality assessment without requiring Content Safety backend.

#### Content Safety Test Behavior

The test gracefully skips when Content Safety service is unavailable:

```python
try:
    result = content_safety_evaluator(query=query, response=response)
    # Process results...
except Exception as e:
    if "404" in str(e) or "not found" in str(e).lower():
        pytest.skip(
            "Azure AI Content Safety service not deployed or not accessible. "
            "Deploy Content Safety to enable this evaluator."
        )
```

This ensures test suite continues running even if Content Safety isn't available.

#### Running Content Safety Tests

**All evaluators (including Content Safety attempt)**:
```bash
pytest tests/evaluation/test_response_quality.py::TestAgentResponseQuality -k "[code]" -v
```

**Only working evaluators (skip Content Safety)**:
```bash
pytest tests/evaluation/test_response_quality.py::TestAgentResponseQuality -k "[code] and not content_safety" -v
```

### Collecting Agent Responses

Use the `agent_runner.py` utility to collect responses for evaluation:

```bash
# Collect responses for design endpoint
python tests/evaluation/agent_runner.py \
    --dataset tests/evaluation/datasets/design_requests.jsonl \
    --endpoint design \
    --output design_responses.jsonl \
    --api-url https://your-api.azurecontainerapps.io

# Collect responses for code analysis endpoint
python tests/evaluation/agent_runner.py \
    --dataset tests/evaluation/datasets/code_analysis_requests.jsonl \
    --endpoint code \
    --output code_responses.jsonl \
    --api-url https://your-api.azurecontainerapps.io

# Batch mode: process all datasets
python tests/evaluation/agent_runner.py \
    --batch \
    --datasets-dir tests/evaluation/datasets/ \
    --limit 5 \
    --api-url https://your-api.azurecontainerapps.io  # Limit to 5 requests per dataset
```

The `agent_runner.py` supports the following endpoints:

| Endpoint Flag | API Path | Dataset File |
|---------------|----------|--------------|
| `--endpoint design` | `/generateDesign` | `design_requests.jsonl` |

### Dataset File Format

**File naming**: `<endpoint>_requests.jsonl`

**Example**: `code_analysis_requests.jsonl`

```json
{"app_id": "50000", "repo_url": "https://github.com/spring-projects/spring-petclinic", "storage_account_name": "publicthisfndrysak6jep", "user_object_id": "3fe36792-20f7-411b-ae89-f9ed833cff05", "resource_group_name": "rg-ai-foundry-standard", "perform_security_scan": true, "test_scenario": "Java code security analysis from a GitHub repository", "expected_status": "accepted", "source_type": "github", "ground_truth_file": "code/codeanalyzer-report-java-001.md", "metadata": {"language": "java"}}
{"app_id": "50000", "repo_url": "https://publicthisfndrysak6jep.blob.core.windows.net/50000/code-analyzer/input/terraform.zip", "storage_account_name": "publicthisfndrysak6jep", "user_object_id": "3fe36792-20f7-411b-ae89-f9ed833cff05", "resource_group_name": "rg-ai-foundry-standard", "perform_security_scan": true, "test_scenario": "terraform code security analysis from a Blob storage", "expected_status": "accepted", "source_type": "blob", "ground_truth_file": "code/codeanalyzer-report-terraform-001.md", "metadata": {"language": "terraform"}}
```

**Important**: Dataset files contain **structured request fields** (app_id, repo_url, etc.), not a `query` field. The `query` parameter used by evaluators is **automatically generated** by `agent_runner.py` from these fields.

**How query generation works:**

When `agent_runner.py` processes requests, it constructs a human-readable query string:

1. **Uses `test_scenario` field if available** (recommended):
   ```python
   query = "Java code security analysis from a GitHub repository"
   ```

2. **Falls back to auto-generated description** if `test_scenario` is missing:
   ```python
   query = "Analyze code for application 50000 from https://github.com/..."
   ```

This auto-generated `query` is then stored in the response JSONL file and used by all evaluators (Relevance, Coherence, Fluency, Similarity).

### Required Fields

| Field | Type | Description |
|-------|------|-------------|
| `app_id` | string | Application identifier (used as container name) |
| `storage_account_name` | string | Azure Storage account name |
| `user_object_id` | string | User's Azure AD object ID |
| `resource_group_name` | string | Azure resource group name |
| `test_scenario` | string | **Recommended** - Human-readable description used as evaluation `query` |
| `ground_truth_file` | string (optional) | Path to baseline file for similarity testing |

### Adding New Test Scenarios

1. **Create dataset entry**:
   ```bash
   echo '{"query": "Your test query", "test_scenario": "Description", "ground_truth_file": "endpoint/baseline-001.md"}' >> tests/evaluation/datasets/<endpoint>_requests.jsonl
   ```

2. **Create baseline** (for similarity tests):
   ```bash
   # Create baseline file at specified ground_truth_file path
   touch tests/evaluation/baselines/<endpoint>/baseline-001.md
   # Add expected response content
   ```

3. **Collect agent responses**:
   ```bash
   python tests/evaluation/agent_runner.py \
       --endpoint <endpoint_name> \
       --dataset tests/evaluation/datasets/<endpoint>_requests.jsonl \
       --output tests/evaluation/responses/<endpoint>_responses.jsonl \
       --api-url https://your-api.azurecontainerapps.io
   ```

4. **Run evaluations**:
   ```bash
   pytest tests/evaluation/test_response_quality.py -k "[<endpoint>]" -v
   ```

## 📝 Baseline Management

Baselines are ground truth responses used for similarity testing.

### Directory Structure

```
tests/evaluation/baselines/
├── code/
│   ├── codeanalyzer-report-java-001.md
│   └── codeanalyzer-report-terraform-001.md
├── design/
│   └── design-001.md
├── asr/
│   └── asr-report-001.md
└── <endpoint>/
    └── <baseline-name>.md
```

### Creating Baselines

**Option 1: Manual Creation**

```bash
# Create directory
mkdir -p tests/evaluation/baselines/<endpoint>

# Create baseline file
cat > tests/evaluation/baselines/<endpoint>/baseline-001.md << 'EOF'
# Expected Response Content

[Your ideal agent response here]
EOF
```

**Option 2: From Actual Response**

```bash
# Run agent to get response
python tests/evaluation/agent_runner.py \
    --endpoint <endpoint> \
    --dataset tests/evaluation/datasets/<endpoint>_requests.jsonl \
    --output tests/evaluation/responses/<endpoint>_responses.jsonl

# Review response in responses file
# If satisfactory, copy to baselines
cp tests/evaluation/responses/<endpoint>_responses.jsonl \
   tests/evaluation/baselines/<endpoint>/baseline-001.md
```

### Baseline Best Practices

✅ **Include complete responses** - Don't truncate or summarize  
✅ **Use descriptive filenames** - `<endpoint>/<scenario>-001.md`  
✅ **Version baselines** - Use `-001`, `-002` suffixes for iterations  
✅ **Document expectations** - Add comments explaining what makes it "ground truth"  
✅ **Update regularly** - Refresh baselines as agent capabilities improve

## 🔧 Troubleshooting

### Common Issues

#### 1. "No collected responses found"

**Problem**: Test skips because response file doesn't exist.

**Solution**:
```bash
# Collect responses using agent_runner.py
python tests/evaluation/agent_runner.py \
    --endpoint <endpoint_name> \
    --dataset tests/evaluation/datasets/<endpoint>_requests.jsonl \
    --output tests/evaluation/responses/<endpoint>_responses.jsonl \
    --api-url https://your-api.azurecontainerapps.io
```

#### 2. "Metrics show as Evaluation_unknown"

**Problem**: `metric_name` field not set in results.

**Solution**: This is now fixed. Each metric is stored with explicit `metric_name` and `score` fields.

#### 3. "Metrics don't appear in Application Insights"

**Problem**: Metrics take 5-10 minutes to appear.

**Troubleshooting**:
```kusto
// Check if metrics are being ingested
customMetrics
| where timestamp > ago(1h)
| where name startswith "Evaluation_"
| take 50
```

**Check verbose logs**:
```bash
# Look for telemetry client logs
pytest ... | grep "TelemetryClient"
pytest ... | grep "Metric:"
```

#### 4. "Similarity test fails"

**Problem**: Response doesn't match baseline closely enough.

**Solutions**:
- **Review baseline**: Is it still current?
- **Check agent output**: Has response format changed?
- **Adjust threshold**: Lower similarity threshold if acceptable (not recommended)
- **Update baseline**: Create new baseline from current response if quality is good

#### 5. "Connection to AI Foundry fails"

**Problem**: Invalid or expired credentials.

**Solution**:
```bash
# Refresh Azure credentials
az login

# Verify connection string
echo $AZURE_EXISTING_AIPROJECT_ENDPOINT

# Test connection
python -c "from azure.ai.projects import AIProjectClient; from azure.identity import DefaultAzureCredential; client = AIProjectClient(endpoint='<your-endpoint>', credential=DefaultAzureCredential()); print('✅ Connected')"
```
