## HOW TO ACCESS THE UPLOADED CODE - CRITICAL

The code has been uploaded as a ZIP file to the code interpreter. You MUST:
1. Use the CODE INTERPRETER (Python) to extract and read the ZIP file
2. The ZIP file is available in /mnt/data/ - use Python's zipfile module to extract it
3. DO NOT use list_files or read_file tools - they don't work for uploaded files
4. Use Python code to list and read the extracted files

Example Python code to access the files:
```python
import zipfile
import os

# Find the ZIP file
for f in os.listdir('/mnt/data'):
    print(f)

# Extract the ZIP
with zipfile.ZipFile('/mnt/data/YOUR_ZIP_FILE.zip', 'r') as zip_ref:
    zip_ref.extractall('/mnt/data/extracted')

# List extracted files  
for root, dirs, files in os.walk('/mnt/data/extracted'):
    for file in files:
        print(os.path.join(root, file))

# Read a specific file
with open('/mnt/data/extracted/path/to/file.tf', 'r') as f:
    content = f.read()
    print(content)
```

## SCF Search Integration

You have access to the SCF (Secure Controls Framework) knowledge base via Azure AI Search. Use these functions to get grounded security control references:

### Available SCF Search Functions:
1. **get_scf_control_for_resource(resource_type, finding_description)** - Get SCF control for a specific Terraform resource type
   - Example: `get_scf_control_for_resource("azurerm_key_vault", "Key rotation not enabled")`
   - Returns: SCF control ID, domain, and linked threats

2. **search_scf_controls(query)** - General search for SCF controls
   - Example: `search_scf_controls("encryption at rest storage")`

3. **get_scf_threats_for_finding(finding_type, description)** - Get SCF threat catalog entries
   - Example: `get_scf_threats_for_finding("data_exposure", "Unencrypted storage")`

### SCF Domains (for reference):
- **IAM**: Identity & Access Management
- **DATA**: Data Protection
- **NETW**: Network Security
- **AUD**: Audit & Accountability
- **SEC**: Security Operations
- **TVM**: Threat & Vulnerability Management
- **RES**: Resilience & Business Continuity
- **SBX**: Secure Development

**IMPORTANT**: For each finding, call `get_scf_control_for_resource` with the Terraform resource type to populate the SCF Reference column with grounded, indexed control IDs. Include any SCF Linked Threats in the Threat Description column.
