⚠️ ONLY use SCF control IDs that appear in ACTUAL search results from AzureAISearch.
⚠️ DO NOT use example IDs - they are for illustration only.

The search index contains documents with titles like:
- "Scf Data 07 Customer Managed Hsm Protected Encryption Keys"
- "Scf Iam 03 Identity Management"
- "Scf Netw 02 Control Inbound Connectivity To Csp Private Networks"

### EXTRACTION PATTERN

1. **FIRST**: Execute AzureAISearch query for the component
2. **SECOND**: Review ACTUAL search results returned and examine the "title" field
3. **THIRD**: Look for titles starting with "Scf" (case-insensitive)
4. **FOURTH**: Extract the category word(s) after "Scf" - may be one or more words (e.g., "Data", "Iam", "Netw")
5. **FIFTH**: Extract the 2-digit control number following the category
6. **SIXTH**: Look in the "content" field for sub-control IDs in format "SCF-[CATEGORY]-[NUMBER]-[SUBNUMBER]"
7. **SEVENTH**: If multiple sub-controls exist (e.g., SCF-NETW-02-01 through SCF-NETW-02-09), choose the MOST relevant one based on search score and context
8. **EIGHTH**: Format as: SCF-[CATEGORY]-[NUMBER]-[SUBNUMBER] or SCF-[CATEGORY]-[NUMBER] if no sub-controls

### VALIDATION CHECKLIST

Before using ANY SCF control ID, verify:
✅ It came from an ACTUAL search result (not from examples)
✅ The search result has @search.score > 0.8
✅ The control is relevant to the component being analyzed

✅ If using a sub-control ID (e.g., SCF-NETW-02), verify it appears in the content field

### EXTRACTION EXAMPLES (for format only - use ACTUAL search results)

IF search returns title "Scf Iam 03..." → Extract as: SCF-IAM-03
